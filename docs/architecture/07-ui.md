# 07 — UI & input

> Every panel reads agent/sim state fresh each frame and is **strictly read-only** —
> the UI never mutates the simulation. The one exception is dialogue, which routes
> player choices back through gameplay systems (recruiting, quests), not by poking
> agent state directly.

## The HUD facade (`js/ui/hud.js`)

`Hud` is the single facade `main.js` talks to. It instantiates every panel + the
dialogue modal, owns the transient DOM readouts (HP fill, hurt flash, the debug/crash
line, player stats, context prompt), and exposes:
- `setWorld(sim, …)` — re-wires every panel to the current `Simulation`, so panels
  survive a world rebuild.
- `render(game, mouseNDC, stageFn)` — per-frame panel refresh, called as the
  `hud.render` stage in the [frame loop](01-sim-spine.md).
- `toggleTab(n)` / `questLog`/`inventory`/… handles for the keymap.

Most panels are **self-injecting**: they create their own CSS + root node at
construction (no `index.html` edits) and toggle with a key. The tab panels can be
dragged out of the left stack (`dragtabs.js`).

| Class Codex | `classCodex.js` | `K` | The emergent [class system](03-rpg-abilities.md): classes in play across all agents (template + procedural), templates, and the player's own classes/levels. |

while preserving click-to-collapse — a tab detaches to `position: fixed` on first drag.

## Dialogue (`js/dialogue/dialogue.js` + `js/ui/dialogueView.js`)

Press `E` near an NPC to talk. `dialogue.js` builds a `DialogueSession` from the NPC's
**beliefs and memories** (so an NPC speaks from what it knows/thinks, including about
the player); `dialogueView.js` renders it as a self-injecting modal with clickable
option buttons. Options like `[Join my party]` (gated on reputation standing) route back
into gameplay systems — this is the one place player UI *drives* the sim, and it does so
through the proper systems ([Party](06-world-dungeons.md), [Quests](05-economy-news.md)),
not by mutating agent fields. Optional LLM-written lines (off by default) come from
[`js/ai/llm.js`](05-economy-news.md); with the LLM off, dialogue is exactly templated.

## Keymap (`js/playerControls.js`)

Held by `PlayerControls` (the global keydown map):

| Key | Action |
| --- | --- |
| `WASD` / `Shift` | move / run |
| LMB-flick / RMB-flick | attack (aim by flick direction) / block |
| `1`–`4` | cast ability slots (`player.abilityList()`, edge-triggered) |
| `G` / `H` | gather from a resource node / drink potion to heal |
| `E` | talk to NPC / use dungeon portal (context action) |
| `F` | pin/unpin the Inspector on the looked-at agent |
| `Q` `B` `K` `Y` `N` `J` | toggle Quest Log / Inventory / Class Codex / Ability Index / Chronicle / Gazette |
| `C` `T` `I` `M` | toggle HUD tabs 1–4 (letters chosen so they never clash with 1-4 casts) |
| `Esc` / `R` | pause (free cursor) / reset world |

## Invariants & gotchas

- **UI is read-only.** Panels read agent/sim state each frame and never write it (except
  dialogue routing through gameplay systems). Keep new panels read-only — a UI write
  would silently corrupt the simulation and break determinism/headless parity.
- **`setWorld` re-wiring.** On world rebuild, panels must be re-pointed at the new
  `Simulation` via `Hud.setWorld`. A panel caching the old `sim` is a stale-world bug.
- **Self-injecting panels** add their own CSS/DOM — follow the existing pattern (mirror
  `questLog.js`/`inventoryPanel.js`) rather than editing `index.html`.
- **Letter toggles dodge the cast keys.** Tab toggles use `C/T/I/M` precisely so they
  don't collide with the `1-4` ability casts — keep new bindings clear of `1-4`.
